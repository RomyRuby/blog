2. display的属性值及其作用 #

none:元素不显示，并从文档流中移除

block：块级元素

inline-block：行内块元素

inline：行内元素

flex：flex布局

table：块级表格

Inherit：继承父元素display

3. display的block、inline和inline-block的区别 #

block：块级元素，独占一行，可以设置width、height、margin、padding

inline：行内元素，不单独一行，设置width、height、和垂直方向上的margin、padding无效

inline-block：行内块，不独占一行，但可以设置width、height、margin、padding

4. 隐藏元素的方法有哪些 #

display：none，文档流中移除，渲染树中不存在，不在页面中占位，不触发事件，不可继承，不可被读屏器读取

visibility：hidden，隐藏，在页面中仍占据位置，但不能触发绑定事件，可继承，可被读屏器读取

opacity：0，透明，在页面中占据位置，可以触发绑定事件

position：absolute，移出可视区域

z-index：负

clip/clip-path：裁剪掉，占位，但不触发事件

transform：scale(0,0)，缩小为0，占位，不响应事件

5. 对盒模型的理解 #

盒模型=content+padding+border+margin

标准盒模型：width=content

IE盒模型（怪异盒模型）：width=content+padding+border

box-sizing：content-box 设置标准盒模型（默认）

box-sizing：border-box 设置IE盒模型

6. CSS3中有哪些新特性 #

选择器、border-radius、text-shadow、gradient、transform、动画（旋转、缩放、倾斜、定位）

8. 两栏布局的实现 #

两栏布局：左边固定，右边自适应

  1. 左边浮动，设置宽度为固定值；右边设置margin-left为左边元素宽度，宽度设置为auto
  2. 左边浮动，设置宽度为固定值；右边设置overflow：hidden触发BFC，BFC区域不会与浮动元素重叠
  3. 父元素设置display：flex，左元素设置宽度，右元素设置flex：1
  4. 父元素设置position：relative，左元素设置position：absolute和宽度，右元素margin-left左元素宽度
  5. 父元素设置position：relative，左元素宽度，右元素position：absolute，left：左元素宽度，top、bottom、right都为0

9. 三栏布局的实现 #

三栏布局：左右固定、中间自适应。

  1. 左右两栏设置为绝对定位，中间设置对应方向大小的margin的值
  2. 利用flex布局，左右两栏设置固定大小，中间一栏设置为flex:1
  3. 左浮动+右浮动，中间放最后，设置两个方向的margin
  4. 圣杯布局：父元素设置左右padding，中间最前左浮动，左边左浮动margin-left：-100%，右元素右浮动margin-left：-右宽
  5. 双飞翼布局：中间元素margin左右，左元素左浮动：left：-100%，右元素左浮动margin-left：-右宽

11. 对Flex布局的理解及其使用场景 #

flex：设置盒模型（flex container）为弹性布局，同时使子元素（flex item）的float、clear、vertical-align失效

flex container有6个属性：

  • flex-direction：决定主轴的方向（flex item的排列方向）
  • flex-wrap：如何换行
  • flex-flow：flex-direction flex-wrap 默认值为 row nowrap
  • justify-content：在主轴上的对齐方式
  • align-items：交叉轴上如何对齐
  • align-content：多根轴线的对其方式

flex item有6个属性：

  • order：排列顺序
  • flex-grow：如果存在剩余空间放大的比例，默认为0，不放大
  • flex-shrink：如果空间不足项目缩小的比例，默认为1，会缩小
  • flex-basis：定义分配空间前，占据主轴空间的大小，默认为auto即项目本身的大小
  • flex：flex-grow flex-shrink flex basis （flex:1=flex: 1 1 0）
  • align-self：单个项目对齐方式

12. 为什么需要清除浮动？清除浮动的方式 #

浮动会引起的问题：

  • 父元素高度坍塌

    解决：

    • 定义父元素height
    • 最后一个浮动元素添加空div或伪元素，设置clear:both
    • 设置父元素overflow:hidden触发BFC

13.使用 clear 属性清除浮动的原理？ #

clear的意思是元素盒子的边不能和前面浮动元素相邻，由于浮动要么是左浮动，要么是右浮动，所以clear:both = clear:left+right，即能清除浮动。

14. 对BFC的理解，如何创建BFC #

BFC是一个独立的布局环境，在BFC容器中不会受外部环境的影响。BFC容器在垂直方向上于文档流排列一致。能解决margin重叠和父元素高度坍塌的问题。

触发BFC的条件：

  • 根元素：body，
  • 元素设置浮动
  • 元素设置绝对定位：position(absolute，fixed)
  • Display:flex/inline-block/table-cell
  • overflow:hidden

15. 什么是margin重叠问题？如何解决？ #

问题：上下两个块级元素（脱离文档流的浮动元素和绝对定位元素除外）垂直方向上，中间的边距会重叠，大小为大的那个边距。

解决：使用BFC

16. position的属性有哪些，区别是什么 #

  • absolute：绝对定位，相对于有定位的第一个父元素设置left\right\top\bottom
  • Relative：相对定位，相对于本身原来的定位设置left\right\top\bottom
  • fixed：固定定位，相对于浏览器设置left\right\top\bottom
  • sticky：粘性定位，relative+fixed，滚动在视口内是relative，滚出去是fixed，实现吸附效果
  • static：默认定位
  • inherit：继承

19. 如何解决 1px 问题？ #

1px问题：在Retina屏幕上1px变得很粗。由window.devicePixelRatio（设备物理像素/css像素）比例来确定。

比如这个比例为2，那解决办法：

  1. 直接写0.5px

  2. 伪元素先放大后缩小，宽高设置为2倍数，但border设置为1，再用scale(0.5)，锁一下两倍

  3. 整个页面缩小2倍

    <meta name:"viewport" content:"initial-scale=0.5,maximum-scale=0.5, minimum-scale=0.5, user-scalable=no" >

20. CSS中可继承与不可继承属性有哪些 #

可继承：大部分与文字字体相关的属性，和visibility

不可继承的：大部分属性，如布局、盒模型的属性都不能继承。

21. link和@import的区别 #

link是html标签，用来加载css等

@import是css里用的，并且是在页面完全加载之后才加载，兼容性也要差一点。

22. 伪元素和伪类的区别和作用？ #

伪元素：在元素前后插入额外的元素，但并不在文档中生成。::before、::after

伪类：某元素的某种状态，不会产生新的元素。:hover、:first-child

23. 常见的图片格式及使用场景 #

GIF：无损压缩，支持动图、透明，缺点是支持色彩少。

JPEG：色彩丰富，缺点是有损压缩、不支持透明。适合用来存储照片。

PNG：无损，支持透明，不支持动画，体积较大。

SVG：无损，支持dom操作，适合做logo、和icon

webp：支持有损和无损，各方面较不错，就是兼容性差。

24. 对 CSSSprites 的理解 #

将很多图片放到一张大图上，用css的background-image，background-repeat，background-position属性进行定位。

能减少http请求很多图片，从而提高性能，多张图合成一张图能减少内存。

缺点是使用和维护困难一点。

22. CSS预处理器/后处理器是什么？为什么要使用它们？ #

预处理器：sass、less，预编译成css，支持变量、函数、循环等。

后处理器：postcss，为css增加一些兼容性处理等。

23. 对line-height 的理解及其赋值方式 #

line-height：一行文本的高度，可以撑开文本高度。

赋值：px、em、纯数字（比例），百分比

24. CSS 优化和提高性能的方法有哪些？ #

  1. 对css文件进行打包压缩
  2. 减少重绘重排
  3. 使用雪碧图
  4. 单一样式语句比组合样式语句执行效率更高
  5. 使用link代替@import（在页面加载完毕之后才加载，可能引起样式闪烁）
  6. 选择器是从右向左找的，会遍历所有元素进行匹配，层级不要嵌套太多
  7. 尽量少使用标签、通配符选择器，而用id、class比较好
  8. 合理利用继承，避免重复设置

30. transition和animation的区别 #

transition是过度动画，只有一个起始帧和一个结束帧

animation是动画，可以用@keyframe多定义几个关键帧

31. 对requestAnimationframe的理解 #

请求帧动画，浏览器在下一次重绘前会调用回调函数，是宏任务，后台运行时不会执行，一个刷新频率内执行一次，并且会合并所有dom操作执行，一般是每秒60帧。

32. 为什么有时候⽤translate来改变位置⽽不是定位？ #

transform会创建一个GPU图层，不会引起页面的重绘和重排，从而提高性能。

33. li 与 li 之间有看不见的空白间隔是什么原因引起的？如何解决？ #

是代码中的空格、回车、tab引起的。

解决：设置字体为0px，字间距，或者把li都写在一行

替换元素：修改某个属性就能被替换的元素，比如图片。

替换元素内容不受css影响，有自己的尺寸，并且都是内联的。

window.innerHeight（浏览器可是高度）+ document.body.scrollTop（滚出的高度）> 元素.offsetTop

36. z-index属性在什么情况下会失效 #

  1. 元素没有设置position
  2. 父元祖position:relative，子元素z-index失效
  3. 设置了float

37. CSS3中的transform有哪些属性 #

transform: rotate | scale | skew | translate

38. 如何根据设计稿进行移动端适配？ #

移动端适配主要有两个维度：

  • 适配不同像素密度， 针对不同的像素密度，使用 CSS 媒体查询，选择不同精度的图片，以保证图片不会失真；
  • 适配不同屏幕大小， 由于不同的屏幕有着不同的逻辑像素大小，所以如果直接使用 px 作为开发单位，会使得开发的页面在某一款手机上可以准确显示，但是在另一款手机上就会失真。为了适配不同屏幕的大小，应按照比例来还原设计稿的内容。

为了能让页面的尺寸自适应，可以使用 rem，em，vw，vh 等相对单位。

39. 元素的层叠顺序 #

正z-index > z-index=0 > inline > float > block > 负z-index > 背景/边框
